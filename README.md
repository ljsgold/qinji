## 功能特性

### 基础功能
- ✅ **菜单交互**：直观的 Web 界面，支持多页面导航
- ✅ **数据存储**：JSON 文件持久化存储，支持数据备份和恢复
- ✅ **函数封装**：模块化设计，代码结构清晰
- ✅ **异常处理**：完善的错误处理和用户输入验证
- ✅ **数据统计**：多维度消费统计分析

### 提高功能
- ✅ **数据可视化**：Plotly图表：消费趋势、分类构成、周规律分析
- ✅ **面向对象**：采用 OOP 设计模式（dataclass）
- ✅ **预算管理**：实时预算执行状态监控与预警

### 创新功能
- ✅ **消费趋势图**：可视化每月消费趋势（折线图）
- ✅ **异常消费检测**：基于Z-score算法的异常消费识别（`/api/anomalies`）
- ✅ **星期分布分析**：按星期分析消费习惯（`/api/weekday-analysis`）
- ✅ **趋势预测**：移动平均算法预测未来消费趋势（`/api/trend-forecast`）
- ✅ **预算预警系统**：80%警告/100%超支双色视觉预警

## 快速开始

### 方法一：双击启动（推荐）

双击运行 `启动系统.bat` 文件即可自动启动后端并打开前端页面。

> 如果你的 Windows 把 `.bat` 文件默认当成文本打开，请在项目目录的命令行中执行 `启动系统.bat` 或 `./启动系统.bat`。

### 方法二：命令行启动

```bash
# 安装依赖
pip install -r requirements.txt

# 启动后端服务
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

启动后，在浏览器访问：`http://127.0.0.1:8000/frontend/index.html`

## API 接口文档

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/records` | 获取消费记录（支持筛选、分页、排序） |
| POST | `/api/records` | 添加消费记录 |
| PUT | `/api/records/{id}` | 更新记录 |
| DELETE | `/api/records/{id}` | 删除记录 |
| POST | `/api/records/clear` | 清空所有记录 |
| GET | `/api/statistics` | 获取统计信息 |
| GET | `/api/summary` | 首页摘要数据 |
| GET | `/api/categories` | 获取分类列表 |
| GET | `/api/export` | 导出CSV |
| POST | `/api/import` | 导入CSV |
| POST | `/api/backup` | 创建备份 |
| GET | `/api/backup/list` | 列出备份 |
| POST | `/api/backup/restore` | 恢复备份 |
| DELETE | `/api/backup/delete` | 删除备份 |
| **GET** | **`/api/anomalies`** | **异常消费检测（加分项）** |
| **GET** | **`/api/weekday-analysis`** | **周规律分析（加分项）** |
| **GET** | **`/api/trend-forecast`** | **趋势预测（加分项）** |

## 界面说明

### 总览页
- 预算追踪卡片（双色预警）

## 注意事项

1. 首次运行会自动加载示例数据（约20条）
2. 所有数据存储在 `data/records.json` 文件中
3. 定期使用"归档"页面创建备份
4. 如需重置数据，可在"归档"页面清空所有记录
5. 预算默认设置为3000元/月，可在 `frontend/index.html` 中修改 `BUDGET` 常量
